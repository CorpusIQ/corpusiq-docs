# Build an AI Knowledge Base — Without Building Anything

Traditional AI knowledge base: choose a platform, upload documents, configure embeddings, set up search, maintain the index, retrain when content changes.

CorpusIQ approach: connect your tools, ask questions.

## The difference

| | Traditional KB | MCP |
|---|---------------|-----|
| Setup | Upload docs, configure vectors | 30-second OAuth |
| Maintenance | Re-index on content changes | Zero — queries live data |
| Data freshness | Days to weeks old | Live, every query |
| Cross-source | Not possible | Built-in |
| Best for | Static policies, wikis | Live business data |

## When to use each

Use a knowledge base for: employee handbook, company policies, onboarding docs.

Use MCP for: revenue, pipeline, customer data, KPIs — anything that changes.

They're complementary. Your handbook shouldn't be in QuickBooks. Your P&L shouldn't be in a wiki.
